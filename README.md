# Кластеризация исполнителей и рекомендательная система

В рамках проекта исследуется поведение алгоритмов кластеризации при группировке музыкантов со схожей аудиторией. На основе выделенных кластеров разрабатывается рекомендательный модуль, помогающий находить похожих артистов и расширять музыкальный кругозор пользователей.

Основная метрика качества: визуальная разделимость кластеров в 2D и оценка топ-10 рекомендаций.

#### Используемые инструменты и библиотеки
* **Анализ данных и предобработка:** `Pandas`, `NumPy`, `OpenPyXL`. `Matplotlib`
* **Машинное обучение:** Scikit-Learn, `KMeans`, `DBSCAN`, `AgglomerativeClustering`, `SpectralClustering`
* **Снижение размерности и метрики:** `TSNE`, `scipy.spatial.distance.cosine`

#### Ход решения

### 1. Подготовка данных & Матричные преобразования
Начальный этап был посвящен формированию корректного признакового пространства для музыкальных исполнителей:
* Выполнено транспонирование исходной матрицы рейтингов для перевода объектов исследования из пользователей в исполнителей.
* Пропуски заполнены нулями, проведена нормализация векторов признаков.

### 2. Базовая кластеризация и Рекомендации
Для разделения исполнителей по латентным музыкальным предпочтениям применен классический подход:
* Алгоритм `KMeans` успешно сгруппировал артистов на 5 стабильных кластеров. Проведена визуализация их центроидов.
* Рекомендательная система: Разработан модуль поиска похожих исполнителей. На основе косинусного расстояния для каждого кластера рассчитывается топ-10 ближайших артистов. К примеру, косинусное расстояние между The Beatles и Coldplay составило ≈ 0.27, что подтвердило их нахождение в одном кластере.

### 3. Визуализация в 2D-пространстве
Для качественного анализа структуры полученных групп многомерные профили исполнителей были спроецированы на плоскость:
* С помощью алгоритма `t-SNE` построена интерактивная двухмерная карта музыкального ландшафта.
* На графике отображены как сами исполнители, так и центроиды их кластеров с соответствующей цветовой кодировкой.

### 4. Эксперименты на нелинейных данных
Чтобы определить границы применимости алгоритмов, проведено их стресс-тестирование на синтетическом нелинейном датасете `make_moons`:
* KMeans: Закономерно не справился с разделением вытянутых нелинейных структур из-за сферической формы кластеризации.
* DBSCAN: Настроен с помощью сетки параметров (Grid Search по 24 комбинациям `eps` и `min_samples`), что позволило идеально отделить «луны» друг от друга.
* Agglomerative Clustering: Проведено сравнительное исследование различных стратегий связывания (`ward`, `complete`, `average`, `single`).
* Spectral Clustering: Реализован перебор параметра `n_neighbors` (от 1 до 19).

## Ключевые выводы
1. Для стандартных профилей популярности `KMeans` в сочетании с косинусным расстоянием дает интерпретируемые и плотные кластеры исполнителей.
2. В условиях нелинейно разделимых данных `DBSCAN` и `Spectral Clustering` (при `n_neighbors` в диапазоне 7–10) демонстрируют превосходство над метрическими методами, безошибочно воссоздавая сложную геометрию распределения.
